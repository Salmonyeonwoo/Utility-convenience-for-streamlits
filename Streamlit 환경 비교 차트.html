Streamlit 환경 비교 분석

로컬 개발(Local Development)과 GitHub/클라우드 배포(Cloud Deployment) 환경의 주요 차이점 비교

✅ 공통점 (코드 일관성)

✔ **주요 코드 구조 (Python):** Streamlit 앱의 로직, 함수, 클래스 등 대부분의 파이썬 코드는 변경 없이 재사용됩니다.

✔ **UI/UX (Streamlit 위젯):** `st.button`, `st.sidebar`, `st.columns` 등 Streamlit 위젯의 동작 및 외관은 동일하게 유지됩니다.

✔ **라이브러리 로드 방식:** `import` 문을 통해 외부 라이브러리를 가져오는 방식은 동일합니다.

❌ 차이점 (환경 및 보안)

항목 로컬 환경 (Local) GitHub/클라우드 (Cloud)
API Key 관리/보안 직접 설정
.env 파일 또는 로컬 .streamlit/secrets.toml에 저장하고 OS 환경 변수로 로드합니다. 		Secrets Injection
Git에는 키를 업로드하지 않고, Streamlit Cloud의 Secrets 관리 시스템을 통해 안전하게 주입받습니다.
실행 환경 및 주소 로컬 머신/터미널
streamlit run app.py 명령으로 실행되며, 주소는 http://localhost:8501입니다. (포트포워딩 필요) 		공개 웹사이트/클라우드
GitHub 저장소와 연동된 Streamlit Cloud에서 호스팅되며, 고유한 HTTPS URL을 가집니다.
의존성 관리 (Dependencies) 자유로움
로컬 Python 환경에 이미 설치된 패키지를 사용하며, requirements.txt의 제약이 적습니다. 		엄격한 통제 ❗
반드시 **모든 종속성**을 명시한 requirements.txt가 있어야 하며, 누락 시 앱 실행이 실패합니다.
데이터 영속성 로컬 디스크 저장
local_db/에 상담 이력, 음성 기록 등 데이터를 영구 저장할 수 있습니다. 		휘발성(Volatile)
로컬 디스크에 저장된 데이터는 세션 종료 또는 재배포 시 사라집니다. (외부 DB 필요)
파일 시스템 접근 로컬 디렉토리 경로(예: C:/Users/...) 접근 가능. 격리된 환경으로, **로컬 디스크에 직접 접근하는 것은 불가능**합니다.

* 포트폴리오를 위해 GitHub에 코드를 업로드할 때는 반드시 .gitignore 파일로 API Key 파일 및 local_db/ 폴더를 제외해야 합니다.
